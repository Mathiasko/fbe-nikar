<task type="auto">
  <name>Task 1: Initialize Astro project with static output</name>
  <files>package.json, astro.config.mjs, src/pages/index.astro, tsconfig.json</files>
  <action>
Create new Astro project in current directory:
1. Run `npm create astro@latest . -- --template minimal --no-git --no-install`
2. Install dependencies: `npm install`
3. Configure astro.config.mjs:
   - Set output: 'static' (default, but explicit)
   - Set build.format: 'directory' for clean URLs
   - Set site to placeholder (will be updated later)
4. Verify static output works with `npm run build` — should create dist/ with HTML files
5. Create basic global CSS file at src/styles/global.css with:
   - CSS reset (minimal)
   - Olive green color variables (--color-primary: #6B7B00, --color-primary-dark: #4A5500, --color-primary-light: #8B9B20)
   - Base typography (system fonts, clean hierarchy)
  </action>
  <verify>npm run build succeeds, dist/ contains index.html, no JS files in output</verify>
  <done>Astro project builds to static HTML, olive green CSS variables defined</done>
</task>

<task type="auto">
  <name>Task 2: Configure i18n routing with SK/EN</name>
  <files>astro.config.mjs, src/i18n/utils.ts, src/i18n/ui.ts</files>
  <action>
Set up Astro's built-in i18n:
1. Update astro.config.mjs with i18n configuration:
   ```
   i18n: {
     defaultLocale: 'sk',
     locales: ['sk', 'en'],
     routing: {
       prefixDefaultLocale: true  // /sk/ and /en/ URLs
     }
   }
   ```
2. Create src/i18n/ui.ts with translation strings for:
   - Navigation items (all 7 pages)
   - Common UI text (language toggle label)
3. Create src/i18n/utils.ts with helper functions:
   - useTranslations(lang) — returns t() function
   - getLocalizedPath(path, lang) — converts paths between languages
   - getCurrentLang(Astro.url) — extracts language from URL
4. Test by creating placeholder src/pages/sk/index.astro and src/pages/en/index.astro
  </action>
  <verify>npm run build creates dist/sk/index.html and dist/en/index.html, URLs work correctly</verify>
  <done>i18n configured, /sk/ and /en/ routes work, translation helpers available</done>
</task>
